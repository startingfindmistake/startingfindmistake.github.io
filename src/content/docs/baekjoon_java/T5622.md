```java
import java.util.Scanner;



public class T5622 {
    public static void main(String[] args){
        Scanner sc = new Scanner(System.in);

        // 첫번째 문자열을 입력받습니다.
        String A = sc.next();
        // 총 시간을 저장할 변수를 0으로 초기화 합니다
        int total_count = 0;

        String[] str = {"","","","ABC","DEF","GHI","JKL","MNO","PQRS","TUV","WXYZ",};


        //문자열에서 각 문자를 꺼내기 위해서 A의 길이만큼 반복하는 for문을 만든다. (i = 0 부터 A.length()까지)
            // 3. A.charAt(i)를 사용해 현재 순서의 문자를 꺼낸다
        for(int i = 0; i < A.length(); i++){
            char initial_A = A.charAt(i);
            // 배열 str를 탐색하는 두번째 for문을 만든다.
            for (int k = 0; k < str.length; k++){
                //`str.indexOf(initial_A)`에서 에러 발생!
                int index = str.indexOf(initial_A);
            }
        }
    }
}

```

`Cannot invoke indexOf(char) on the array type String[]`에러가 발생했다.

**오류설명**
* 컴파일러가 `String[]`(문자열 배열) 타입의 객체에서 `indexOf(char)`라는 메서드를 찾을 수 없을 때 발생하는 전형적인 컴파일 에러 입니다.

## 배열(Array)의 멤버 구성 (JLS 규격)
Java 언어 명세의 **JLS 10.7장(Array Members)**에 따르면, Java에서 배열 타입(예: `String[]`)이 가질 수 있는 멤버는 다음과 같이 엄격하게 정의되어 있습니다.
* 배열의 크기를 나타내는 `public final` 필드인 `length`
* 배열을 복제하는 `pulbic` 메서드인 `clone()`
* 모든 클래스의 최상위 부모인 `java.lang.Object`클래스로부터 상속받은 메서드들 (예 `equlas()`, `hashCode()`, `toStirng()`등)




**에러 해결**

```java
for (int k = 0; k < str.length; k++){
                int index = str[k].indexOf(initial_A);
            }
```

**그런데 문제를 해결하기 위해서는 다른 방식을 생각해야 한다.**
1. 문자를 찾았을때 해당하는 인덱스의 값을 알아야 하고
2. 못 찾았을때는 반복문을 빠져나와 다른 문자열 배열의 다른 인덱스에 문자가 있는지도 확인해야 한다.

그러면 문자을 찾았을때와 못 찾았을때를 구분하기 위해서 `if`문을 사용해볼 예정이다.

그런데 여기서 하나 알아야 할게 `indexOf(String str)`의 특징이다.
이 메서드의 특징은
* 반환되는 인덱스는 `this.startsWith(str, k`)가 참(`true`)일 경우 가장 작은 정수 `k`의 값을 제공한다.
* 해당 부분 문자열이 존재하지 않으면 `-1`을 반환합니다.

```java
// 1. 현재 탐색 중인 문자열(str[k])에 initial_A가 포함되어 있는지 확인한다.
 if(str[k].indexOf(initial_A) != -1){
     // 2. 포함되어 있다면, 인덱스 k(걸리는 시간)만큼 total_count에 더한다.
    total_count += k;
 }
```
