## 마크다운 공백 자동 제거 현상

> 1. 원인에 대해서 알아본다.
> 2. 공식문서에서 힌트를 찾아본다.
> 3. 해결방법을 알아보자

### 원인에 대해서 알아본다

안티그래비티(Google Antigravity)에서 마크다운으로 작성할때 다음줄로 넘어가기위해 스페이스 두 번을 입력하고 다음줄로 넘어갔는데 블로그에서는 다음줄로 명령이 안된다.

가장 큰 원인은 에디터의 **'저장 시 후행 공백 제거(Trim Trailing Whitespace)'** 설정 때문이다.

### 공식 문서에서 힌트를 찾아보자

---

Antigravity 공식 문서의 시작 부분을 보면 에디터의 태생에 대해 이렇게 명시되어 있다.
_"Antigravity forks open-source Visual Studio Code (VS Code) foundation but radically alters the user experience to prioritize agent management over text editing"_

Antigravity는 오픈소스인 visual Stduio Code(VS Code) 기반을 포크(파생)하여 만들어졌으나, 텍스트 편집보다 에이전트 관리를 우선시하도록 사용자 경험을 근본적으로 변경했습니다.

</br>
</br>

즉 여기서 알수 있는 내용에서 중요한 부분은

* **VS Code의 텍스트 편집 기능 상속을 한다.**

공식 문서에 선언된 것처럼 Antigravity의 핵심 코딩 화면(Editor)은 VS Code를 뿌리로 두고 있으며,

따라서 코드를 작성하고 저장할 때 자동도으로 작동하는 '불필요한 공백 지우기' 같은 기본 포매팅(Formatting) 규칙 역시 VS Code의 방식을 그대로 따라간다고 생각할수 있다.

### 해결방법(Solution)
>
>보통 코딩을 할때는 줄 끝에 남는 불필요한 스페이스를 지우는게 코드를 깔끔하게 유지하는 방법이라서,
에디터나 포매터(Formatter, ex:Prettier)가 자동으로 줄 끝에 남는 스페이스를 지우는 기능이 활성화 되어 있습니다.

하지만 마크다운에서는 **스페이스 두 번**이 '줄바꿈(Line break)' 역활을 하는데,
이것을 에디터가 일반적인 코드의 '불필요한 공백'으로 오해해서 저장할때 사라지는 현상이 발생한다.

**1. 마크다운 파일에서만 공백 제거 설정 끄기**
Antigravity 설정에서 마크다운을 작성할 때만 공백을 지우지 않도록 예외 처리르 해주면 된다.

1. 안티그래비티에서 설정 창을 연다.
    * 오른쪽 상단에 톱니바퀴 모양을 클릭하면 된다.
    * `Eiditor Settings`를 클릭한다.

2. `Edit in Settings.json` 검색해서 클릭한다.
    * 파란색으로 표시될것이다.

3. 파일 안의 중괄호 `{}` 사이에 아래 코드를 작성하자

```json
"[markdown]": {
    "files.trimTrailingWhitespace": false
}
```

이 코드의 뜻은 **'마크다운 파일에서만 후행 공백제거 옵션을 끄겠다.'** 이다.

</br>
</br>

**2. 스페이스 두 번 대신 다른 줄바꿈 문법 쓰기**
설정을 건드리기 귀찮거나, 포매터 충돌을 아예 피하고 싶다면  
마크다운의 다른 줄바꿈 방식을 쓰는 것도 좋다.

* **백슬래스(`\`)**: 문장 끝에 `\`를 입력하고 엔터를 치면 줄바꿈이 된다.

* **HTML태그 사용:** 문장 끝에 `<br>`을 입력하면 줄바꿈이 된다.

>여기서 주의할점은 가끔은 백슬래시(`\`)가 줄바꿈으로 인식이 안되고 출력이 될수 있다.
