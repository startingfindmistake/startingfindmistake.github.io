## 먼저 알아야 할 사전 지식

* **배열(Array)**: 동일한 데이터 타입의 값들을 하나의 연속된 공간에 묶어서 저장하는 자료구조 입니다.
    * `new int[4]`는 정수 (`int`) 4개를 저장할 수 있는 공간을 메모리에 생성하라는 의미 입니다.
    * 배열의 인덱스는 `0`부터 시작합니다. (길이가 4라면 인덱스는 0, 1, 2, 3)
    * 배열의 길이는 `배열 이름.length` 속성을 통해 알 수 있습니다.

</br>

* 메서드 (Method)와 반환 (Return)**: 특정 작업을 수행하는 코드의 블록 입니다.
    * `marr()` 메서드는 `static int[]`라는 반환 타입을 가집니다.  
    이는 메서드가 실행을 마치고 나면 정수형 배열(`int[]`)을 결과값으로 돌려준다(return)는 뜻입니다.

</br>

* 참조 변수와 메모리(Reference Variables): Java에서 배열은 '객체' 취급을 받습니다.
    * `int temp[]`나 `int iarr[]` 같은 변수에는 배열의 실제 데이터 (0, 1, 2, 3)이 직접 저장되는 것이 아니라, 데이터가 저장된 메모리 주소(참조값)이 저장됩니다.


```java
public class java_test1 {
    static int[] marr() {
        int temp[] = new int[4];
        for(int i = 0; i < temp.length; i++)
            temp[i] = i;
        return temp;
    }

    public static void main(String[] args) {
        int iarr[];
        iarr = marr();
        for(int i = 0; i < iarr.length; i++)
            System.out.print(iarr[i] + " ");
    }
}
```


#### 1단계: `main` 메서드 시작

```java
public static void main(String[] args) {
    int iarr[];
}
```

* 가장 먼저 `int iarr[];`가 선언됩니다.  
이는 정수형 배열의 주소를 담을 수 있는 참조 변수 `iarr`를 만든 것입니다.  
아직 어떤 배열과도 연결되지 않은 빈 상태입니다.


</br>
</br>

#### 2. `marr()` 메서드 호출

```java
iarr = marr();
```

* `iarr`에 값을 할당하기 위해 `marr()` 메서드를 호출합니다.  
프로그램의 실행 흐름이 `marr()` 메서드 내부로 이동합니다.

</br>
</br>

#### 3. `marr()` 메서드 내부 - 배열 생성

```java
    static int[] marr() {
        int temp[] = new int[4];
    }
```
* `temp`라는 이름의 정수형 배열 참조 변수를 만듭니다.
* `new int[4]`를 통해 메로리(Heap영역)에 4개의 정수가 들어갈 공간을 생성하고,  
그 메로리 주소를 `temp`에 저장합니다.  
이때 배열의 초기값은 모두 `0`입니다. (`{0, 0, 0, 0}`)

</br>

#### 4단계: `marr()` 메서드 내부 - 배열 값 초기화(for문)

```java
for(int i = 0; i < temp.length; i++)
    temp[i] = i;
```
* `temp.length`는 배열의 길이인 `4`입니다. 반복문이 `i=0`부터 `i=3`까지 4번 실행됩니다.
    * `i = 0` 일 때: `temp[0] = 0;`
    * `i = 1` 일 때; `temp[1] = 1;`
    * `i = 2` 일 때; `temp[2] = 2;`
    * `i = 3` 일 때; `temp[3] = 3;`

* 반복문이 끝나면 `temp` 배열은 `{0, 1, 2, 3}`의 값을 가지게 됩니다.

</br>
</br>

#### 5단계: 배열 반환(return)
```java
        return temp;
    }
```
* 완성된 배열을 가리키는 주소값(`temp`가 가지고 있는 값)을 메서드를 호출했던 곳으로 반환합니다.

</br>
</br>

#### 6단계: 반환된 배열을 `iarr`에 저장

```java

    // main 메서드로 돌아옴
    iarr = marr();
```
* `marr()`메서드가 반환한 주소값이 `iarr`변수에 저장됩니다.
* 이제 `main` 메서드의 `iarr` 변수와 `marr` 메서드에서 만들었던 `temp` 변수는 **메모리 상의 동일한 배열** `{0, 1, 2, 3}`을 가리키게 됩니다.


</br>
</br>

#### 7단계: 결과 출력(main의 for문)
```java
    for(int i = 0; i < iarr.length; i++)
        System.out.print(iarr[i] + " ");
    }
```
* `iarr.length` 역시 `4` 입니다.  
반복문을 통해 `iarr` 배열의 요소를 처음부터 끝까지 순회하며 출력합니다.
* `print()` 메서드는 줄바꿈 없이 출력하므로, 각각의 숫자 뒤에 공백(" ")을 붙여서 화면에 출력합니다.

</br>
</br>

#### 최종 실행 결과
```plaintext
0 1 2 3
```

```java
class a {
    static int c = 10;
}

class b{
    int x = a.c + 10;
}

public class java_test2 {
    public static void main(String[] args) {
        b t = new b();
        System.out.print(t.x);
    }
}
```

#### 미리 알아야할 부분

1. `static` 변수(클래스 변수)
* 변수 앞에 `static`이라는 키워드가 붙은 변수입니다.
* 객체(인스턴스)를 생성(`new`)하지 않아도, 클래스 이름으로 직접 접근하여 사용할 수 있습니다.
* 프로그램이 시작될 때 메모리에 딱 하나만 만들어지며, 모든 객체가 공유합니다.
* 코드 예시: `a.c`(클래스`a`의 변수 `c`를 가져옴)

</br>

2. 인스턴스 변수 (일반 멤버 변수)
* `static`가 붙지 않은 일반적인 변수 입니다.
* 반드시 객체를 생성(`new`)해야만 메모리에 만들어지고 사용할 수 있습니다.
* 코드예시: 클래스 `b` 안에 있는 `int x`

</br>
</br>

3. 클래스와 객체 생성(`new` 키워드)
* 클래스는 붕어빵 틀(설계도)이고, 객체는 그 틀로 찍어낸 붕어빵(실체)입니다.
* 설계도에 불과한 클래스를 실제로 메모리에 올려 사용하려면 `new` 키워드를 사용해야 합니다.(예: `new b()`)

</br>
</br>

4. `main`메서드
* 자바 프로그램이 실행될 때 가장 먼저 시작되는 출발점 입니다.  
프로그램은 항상 `main` 메서드의 첫 줄 부터 끝 줄까지 순서대로 실행됩니다.

</br>
</br>


#### 1. 프로그램 시작
```java
public static void main(String[] args) {
```
* 자바 프로그램이 실행되면 가장 먼저 java_test2클래스 안에 있는 `main`메서드를 찾아 실행을 시작합니다.

#### 2. `static`변수 메모리 적재
```java
class a {
    static int c = 10;
}
```
* 프로그램이 실행되면서 `static`이 붙은 변수들은 미리 메모리에 올라갑니다.
* 이때 클래스 `a`의 변수 `c`가 메모리에 생성되고, 10이라는 값이 저장됩니다.  
이제 언제 어디서든 `a.c`라고 부르면 10을 꺼내 쓸 수 있습니다.


#### 3. 객체 생성
```java
b t = new b();
```
* `main`메서드의 첫 줄이 실행됩니다.
* `new b()`를 통해 클래스 `b`의 모양대로 메모리에 새로운 객체(인스턴스)를 만듭니다.  
    * `class b`안에 `new b()`를 하지 않는 것이 정상이다.
        * 클래스는 '설계도'일뿐  
        `class b { ... }` 라고 작성한 것은 컴퓨터에게 "앞으로 `b`란느 객체는 이렇게 생겼어" 라고 설계도를 보여준 것이지, 실제로 물건을 만든 게 아닙니다.
        * 설계도 안에서 제품을 만들 순 없다  
        자동차 설계도(클래스 b)안에 진짜 자동차(객체)가 들어갈 수는 없겠죠? 설계도는 가만히 두고, 나중에 이 설계도가 진짜로 필요할 때 공장(`main`메서드)에서 `new b()` 명령을 내려 진짜 자동차를 찍어내는 것입니다.

    * **이렇게 나중에 `main`메서드에서 `new`를 생성하는 이유**  
    프로그램이 시작하자마자 모든 객체를 다 만들어두면 메모리가 부족해질수 있습니다.  
    그래서 설계도(클래스)만 미리 준비해 두고, 프로그램이 실행되다가 "**아, 이제 진짜`b`객체가 필요하네?"라고 하는 시점 (`main`메서드 안 등)에 `new`를 사용해 만드는 것**이 자바의 정석 입니다.  

    * 만약 new를 쓰지 않으면 크게 두가지 상황으로 나뉩니다.
        * **상황 A:`static`이 없는 일반 변수(`int x`)**
            * new를 하지 않으면 이 변수는 세상에 좆냊하지 않는 상태가 됩니다. 설계도에만 적혀있을 뿐, 실제 메모리에 만들어지지 않았기 때문입니다.
            * 따라서 `new`없이 `t.x`라고 쓰려고 하면 컴퓨터가 설계도만 보고는 값을 읽을 수 없어! `new`로 먼저 객체부터 만들어!" 라며 빨간줄을 띄우고 실행조차 해주지 않습니다.  
            클래스 b 내부에서도 `new` 없이는 사용할수 없습니다.

        * **상황 B: `static`이 붙은 클래스 변수 (`static int c`)**
            * 앞서 클래스 `a`에 있던 `static int c` 기억하시나요? `static`이 붙으면 `new`를 하지 않아도 프로그램이 켜질 때 컴퓨터가 알아서 메모리에 딱 만들어줍니다.
            * 그래서 `new a()`를 하지 않고도 `a.c`라고 치면 바로 10이라는 값을 가져와서 쓸 수 있었던 것입니다.

* 앞에 있는 `b`의 역활: "어떤 모양의 데이터를 담을 것인가? (자료형/ 타입)
    * 맨 앞에 적힌 `b`는 자료형(Data Type)을 의미합니다. (여기서는 우리가 만든 클ㄹ래스 이름 입니다)
    * 자바(java) 컴퓨터에게 "지금부터 내가 변수를 하나 만들 건데, 이 변수에는 `b`클래스 설계도대로 만든 객체만 담을 거야! 라고 미리 규칙을 정해주는 것입니다.
    * 숫자만 담는 변수를 만들 때 `int`를 쓰는 것(`int x = 10;`)과 완전히 같은 원리 입니다.  
    여기서 `int` 대신 우리가 직접 만든 클래스 `b`를 자료형으로 쓴 것입니다.

</br>

* 이 객체의 이름을 `t`라고 지어줍니다.


</br>

#### 4. 인스턴스 변수 `x`초기화 (값계산)
```java
class b {
    int x = a.c + 10;
}
```
* 3단계에서 `b`객체가 만들어질 때, `b`클래스 안에 정의된 변수 `x`도 함께 만들어지며 값을 계산합니다.
* `a.c`를 호출했습니다. 앞서 2단계에서 `a.c`는 10으로 메모리에 저장되어 있었습니다.
* 따라서 식은 `int x = 10 + 10;`이 됩니다.
* 결과적으로 방금 생성된 객체 `t` 내부의 `x` 변수에는 20이라는 값이 저장됩니다.

#### 5.결과 출력
```java
System.out.print(t.x);
```
* `main` 메서드의 두 번째 줄이 실행됩니다.
* 객체 `t` 안에 있는 변수 `x`의 값을 읽어와 화면에 출력하라는 명령 입니다.
* `t.x`의 값은 20이므로, 콘솔 창에 20이 출력되고 프로그램이 정상적으로 종료 됩니다.

</br>

**그런데 만약에**
```java
System.out.print(t);
```
이렇게 출력을 한다면 어떻게 될까?

`System.out.print(t);`를 실행하면 자바는 내부적으로
1. 변수 `t`를 그대로 출력하라고 하네? 이 녀석은 일반 숫자가 아니라 '객체' 잖아."
2. "객체를 그대로 문자열로 표현해야 하니까, 이 객체의 기본 정보(클래스 이름과 식별 번호)를 조합해서 보여줘야겠다."
3. 그 결과, 콘솔 창에는 대략 아래와 같은 형태의 문자열이 출력 됩니다.

```plaintext
b@15db9742
```

#### `t`와 `t.x`의 차이점
* `t.x`: " 객체 `t`안에 있는 변수 `x`의 값"
    * `t`라는 객체 내부의 저장 공간 중 이름이 `x`인 곳에 접근하라는 뜻입니다.
    * 우리가 앞서 계산했듯, 이 공간에는 **20**이라는 정수(int) 데이터가 들어있습니다. 따라서 20이 출력됩니다.

* `t`:"객체 `t` 그 자체(메모리상의 주소 정보)"
    * `t`는 `b`클래스의 모양으로 만들어진 덩어리(객체)전체를 가리킵니다.
    * 자바에서 객체를 담고 있는 변수 (`t`)를 직접 출력하려고 하면, 자바는 이 객체 안에 있는 값(x)을 알아서 꺼내주지 않습니다. 대신 이 객체가 컴퓨터 메모리의 어디에 저장되어 있는지에 대한 정보(주소값과 유사한 해시코드)를 문자열로 만들어서 보여줍니다.
