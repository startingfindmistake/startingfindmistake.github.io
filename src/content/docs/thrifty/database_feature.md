## MongoDB vs SQL 데이터베이스(PostgreSQL, MySQL, MariaDB)
---
SQL데이터베이스(RDBMS)느뉴 정형 데이터 처리에 최적화되어 있습니다.  
MongoDB는 데이터의 유연성과 확장성에 강점이 있씁니다



#### 데이터 모델 및 스키마
* **SQL**: 데이터가 행(Row)과 열(Column)로 구성된 테이블에 저장되며, 엄격하게 정의된 스키마를 요구 합니다.  
구조화된 데이터에 적합합니다.

* **MongoDB**: JSON 형태의 문서로 저장되므로, 비정형 또는 반정형 데이터를 변환 없이 기본 형식 그대로 수집하고 저장할 수 있습니다.

#### 확장성 (Scalability)
* **SQL**: 주로 하드웨어 성능을 높이는 수직적 확장(Scale-up)에 의존합니다. 수평적 확장이 어렵고 구조적으로 병목 현상이 발생할 수 있습니다. (예: MariaDB는 확장을 위해 Spider 등 별도의 엔진이 필요합니다.)

* **MongoDB**: 서버를 추가하여 데이터를 분산시키는 수평적 확장(Scale-out)을 기본적으로 제공하므로, 대용량 데이터와 클라우드 인프라에 훨씬 적합합니다.


#### 쿼리 언어 및 드라이버
* **SQL**: 데이터를 조회하고 다루기 위해 표준화된 SQL(Structured Query Language)을 사용합니다.
* **MongoDB**: 개발자가 사용하는 프로그래밍 언어(Java, Python, Node.js 등)에 자연스럽고 관용적인(Idiomatic) 메서드와 함수로 구현된 MongoDB Query API를 사용합니다.

## MongoDB vs 인메모리/키-값 저장소 (Redis)
---
Redis와 같은 인메모리 데이터베이스는 캐싱과 극단적인 속도가 필요할 때 주로 사용됩니다.


#### 저장 방식 및 비용

* **Redis**: 모든 데이터가 RAM(메모리)에서 액세스되므로 속도가 매우 빠르지만, RAM은 디스크 저장소에 비해 비용이 비싸 대규모 데이터 세트를 저장하기에는 비용 효율성이 떨어집니다.  

* **MongoDB**: 디스크 기반 스토리지 엔진(WiredTiger)을 사용하며 내부 캐시를 통해 성능을 최적화합니다. 훨씬 더 크고 복잡한 데이터를 경제적으로 저장하고 관리할 수 있습니다. (엔터프라이즈 버전에서는 인메모리 스토리지 엔진도 제공합니다.)


## 데이터 구조
* **Redis**: 데이터를 단일 컬렉션으로 취급하며, 고유한 '키'와 '값'의 쌍으로 데이터를 저장하고 검색합니다.

* **MongoDB**: 관련된 문서를 '컬렉션(Collection)'으로 그룹화하여 보다 복잡하고 계층적인 데이터 구조를 관리할 수 있습니다.
