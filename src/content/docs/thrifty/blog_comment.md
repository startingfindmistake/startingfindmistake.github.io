정적 블로그나 개인 웹사이트를 운영하다 보면 방문자와 소통하기 위한 **댓글 시스템**이 필요해집니다. 워드프레스나 티스토리 같은 가입형 블로그와 달리, 정적 블로그는 자체 데이터베이스가 없기 때문에 외부 서비스를 연동하여 댓글 기능을 구현해야 합니다.

이 글에서는 블로그에 적용할 수 있는 다양한 댓글 시스템들을 특징별로 분류하고, 각각의 장단점과 작동 원리를 소개합니다. 본인의 블로그 성격과 개발 환경에 가장 잘 맞는 댓글 시스템을 선택해 보세요!

## Github 기반

---

### 1. Giscus

GitHub Discussions 기능을 사용합니다. 속도가 매우 빠르고 테마 대응(다크모드)이 잘 작동합니다. 가장 모던하고 많이 사용됩니다.

- **장점(Pros)**:`Issues` 탭을 오염시키지 않고 순수하게 커뮤니케이션 용도로 설계된 `Discussions`를 활용한다는 것이 아키텍처상 가장 큰 장점입니다.  
React, Vue, 그리고 Astro와 같은 모던 프레임워크를 위한 컴포넌트를 공식 지원하여 연동이 매우 매끄럽습니다.

</br>

- **단점(Cons)**: 방문자가 반드시 GitHub 계정으로 OAuth 인가(로그인)를 거쳐야 합니다.

>##### 간단 작동 원리
>
>- GitHub의 GraphQL API와 Discussions 기능을 사용합니다.  
브라우저에서 Giscus스크럽트가 로드되면, 설정된 GitHub저장소의 Discussion 카테고리에서 현재 페이지의 URL이나 제목과 일치하는 스레드를 찾습니다.  
만약 없다면 Giscus 봇이 새로운 스레드를 생성합니다.
>

- [Giscus 사이트 바로가기](https://giscus.app/ko)

### 2. Utterances

GitHub Issues 기능을 기반으로 하는 가볍고 심플한 댓글 시스템입니다. Giscus의 원형이 되는 프로젝트이기도 합니다.

- **장점(Pros)**: 설정이 매우 직관적이고 간단합니다. 광고가 없으며 완전한 오픈소스입니다. 페이지 로딩 속도에 영향을 거의 주지 않을 만큼 가볍습니다.

</br>

- **단점(Cons)**: `Issues` 탭을 커뮤니케이션 용도로 사용하므로 실제 버그 트래킹 용도의 이슈와 섞일 수 있습니다. Giscus에 비해 다양한 기능을 제공하지 않습니다. 방문자가 반드시 GitHub 계정으로 로그인해야 합니다.

>##### 간단 작동 원리
>
>- GitHub 이슈 검색 API를 사용하여 페이지의 URL, 제목 등을 기반으로 이슈를 찾습니다.  
>일치하는 이슈가 없으면 Utterances 봇이 새로운 이슈를 생성합니다. 사용자가 댓글을 달면 해당 이슈의 코멘트로 등록됩니다.
>

- [Utterances 사이트 바로가기](https://utteranc.es/)

### 3. Gitalk

GitHub Issues 기능을 기반으로 작동하며, React로 만들어진 모던한 댓글 컴포넌트입니다.

- **장점(Pros)**: UI가 깔끔하고 마크다운을 완벽하게 지원합니다. 단축키(Cmd/Ctrl + Enter)를 지원하여 빠른 댓글 작성이 가능합니다.

</br>

- **단점(Cons)**: 초기 세팅 시 GitHub OAuth Application을 등록하고 Client ID와 Client Secret을 발급받아야 하는 번거로움이 있습니다. Utterances와 마찬가지로 `Issues` 탭을 사용하며, GitHub 로그인 필수입니다.

>##### 간단 작동 원리
>
>- GitHub OAuth App을 통해 인증하고 GitHub REST API를 사용하여 이슈를 생성하고 코멘트를 관리합니다.  
>각 페이지마다 고유한 ID(보통 URL)를 레이블로 지정하여 이슈와 페이지를 연결합니다.
>

- [Gitalk 사이트 바로가기](https://gitalk.github.io/)

## 가입없이 누구나 쉽게 댓글달수 있는

---

### 1. Cusdis

개인 정보 보호를 중시하는 가볍고 심플한 오픈소스 댓글 시스템입니다. 무거운 시스템(Disqus 등)의 훌륭한 대안으로 설계되었습니다.

- **장점(Pros)**: 방문자의 가입이나 로그인이 전혀 필요하지 않으며, 쿠키를 사용하지 않아 프라이버시 친화적입니다. 매우 가볍고(gzipped 5kb) 로딩이 빠릅니다. 이메일 알림을 지원합니다.

</br>

- **단점(Cons)**: 익명 댓글이 가능하므로 스팸 관리에 신경 써야 할 수 있습니다. 소셜 로그인 등 다양한 부가 기능은 제공하지 않습니다.

>##### 간단 작동 원리
>
>- Cusdis 클라우드(무료)를 사용하거나 직접 서버에 셀프 호스팅할 수 있습니다.  
>사용자가 댓글을 작성하면 서버로 전송되고, 블로그 소유자는 대시보드나 이메일 알림을 통해 댓글을 승인하거나 거절할 수 있는 모더레이션 기능을 사용합니다.
>

- [Cusdis 사이트 바로가기](https://cusdis.com/)

### 2. Waline

Valine에서 파생된 댓글 시스템으로, 자체 백엔드가 포함된 완전한 기능의 댓글 시스템입니다.

- **장점(Pros)**: 로그인 없이 댓글 작성이 가능하며, 마크다운, 이모지, 수학 수식(KaTeX), 글자 수 제한, 스팸 필터링(Akismet) 등 매우 풍부한 기능을 제공합니다. Vercel 등을 통해 쉽게 무료 배포가 가능합니다.

</br>

- **단점(Cons)**: 기능이 많은 만큼 세팅(데이터베이스 연동, 백엔드 배포)이 다른 서버리스 도구들에 비해 조금 복잡할 수 있습니다.

>##### 간단 작동 원리
>
>- 클라이언트 측 자바스크립트와 자체 백엔드(Node.js 등) 및 데이터베이스(LeanCloud, MongoDB, Vercel Postgres 등)가 통신합니다.  
>사용자가 댓글을 달면 백엔드 서버에 저장되고, 페이지 로드 시 서버에서 댓글 데이터를 불러와 렌더링합니다.
>

- [Waline 사이트 바로가기](https://waline.js.org/)

## 소셜 기반 & 대형 서비스 (다양한 로그인 지원)

---

### 1. Disqus

전 세계적으로 가장 널리 사용되는 상용 블로그 댓글 시스템 중 하나입니다.

- **장점(Pros)**: Facebook, Twitter, Google 등 다양한 소셜 계정 로그인을 지원하며, 강력한 스팸 필터와 모더레이션 도구를 제공합니다. 사용자 풀이 넓어 다른 블로그의 Disqus 사용자가 쉽게 댓글을 남길 수 있습니다.

</br>

- **단점(Cons)**: 스크립트 용량이 크고 무거운 요청이 많아 페이지 로딩 속도를 크게 저하시킵니다. 무료 버전의 경우 광고가 삽입되며, 사용자의 활동 데이터를 추적하여 프라이버시 이슈가 꾸준히 제기됩니다.

>##### 간단 작동 원리
>
>- 페이지에 Disqus 자바스크립트 위젯을 임베드합니다.  
>위젯은 Disqus 서버와 통신하여 iframe 형태로 댓글 UI를 렌더링하고, 사용자의 인증 및 댓글 데이터를 Disqus 중앙 서버에서 관리합니다.
>

- [Disqus 사이트 바로가기](https://disqus.com/)

### 2. Facebook Comments

Facebook에서 공식적으로 제공하는 소셜 플러그인 댓글 시스템입니다.

- **장점(Pros)**: Facebook 사용자가 로그인 상태를 유지한 채로 즉시 댓글을 달 수 있어 접근성이 높습니다. 사용자가 댓글을 자신의 Facebook 피드에 공유할 수 있어 바이럴 효과를 기대할 수 있습니다.

</br>

- **단점(Cons)**: 디자인 커스터마이징이 매우 제한적(다크모드 지원 미비 등)입니다. 철저하게 Facebook 생태계에 종속되며, 페이스북 계정이 없는 사용자는 댓글을 달기 어렵습니다.

>##### 간단 작동 원리
>
>- Facebook의 SDK 스크립트를 페이지에 추가하고, 특정 URL과 연동된 소셜 플러그인 컴포넌트를 렌더링합니다.  
>모든 데이터와 인증 처리는 Facebook의 인프라 내에서 처리되며, iframe 형태로 페이지에 표시됩니다.
>

- [Facebook Comments 사이트 바로가기](https://developers.facebook.com/docs/plugins/comments/)

## 그래서 나의 선택은??

---

### 💡 왜 Giscus를 선택했을까?

수많은 댓글 시스템 중에서도 제 블로그에 **Giscus**를 선택한 이유는 다음과 같습니다.

### 1. 개발자 친화적인 환경 (GitHub 생태계)

제 블로그의 주 방문자 층은 개발자이거나 개발에 관심이 있는 분들일 확률이 높습니다. 따라서 대부분이 이미 GitHub 계정을 가지고 있기 때문에, 별도의 회원가입 없이 기존 계정으로 쉽게 접근하고 댓글을 달 수 있다는 점이 큰 매력으로 다가왔습니다.

### 2. 깔끔한 UI와 완벽한 다크모드 지원

요즘 웹사이트에서 다크모드는 필수적인 기능입니다. Giscus는 테마 전환에 매우 유연하게 대응하며, 특히 Astro와 같은 최신 프레임워크와 이질감 없이 어우러지는 모던한 디자인을 제공합니다.

### 3. 광고 없는 가볍고 안전한 오픈소스

상용 서비스(Disqus 등)에서 흔히 발생하는 무거운 스크립트로 인한 페이지 로딩 속도 저하 문제나 원치 않는 광고 삽입에서 자유롭습니다. 또한, 방문자를 추적하는 기능이 없어 프라이버시 측면에서도 안전합니다.

### 4. Issue 탭과의 분리 (Discussions 활용)

Utterances나 Gitalk과 달리, Giscus는 GitHub의 `Discussions` 기능을 사용합니다. 덕분에 실제 코드 버그나 프로젝트 관리 용도로 사용하는 `Issues` 탭이 댓글로 오염되지 않아 레포지토리를 훨씬 깔끔하게 유지할 수 있습니다.

> **결론적으로 빠른 속도, 세련된 디자인, 그리고 개발자 블로그에 최적화된 편의성을 모두 갖춘 Giscus가 제 블로그 성격에 가장 적합한 댓글 시스템이라고 판단했습니다!**
