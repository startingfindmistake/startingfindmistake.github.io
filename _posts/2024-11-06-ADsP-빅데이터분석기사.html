1 과목: 데이터 이해

01 데이터의 정의

데이터는 객관적 사실이나 존재적 특징과 동시에 추론, 예측, 추정을 위한 근거로 기능하는 의미적 특성을 의미한다.

데이터는 다른 객체와의 상호관계에 따라 가치를 갖는다.

출제 유형

  • 객관적 사실로서 개별 데이터는 중요하지 않음을 의미한다.
  • 객관성 오답 문제

02 데이터의 유형

정성적 데이터 (Qualitative data): 의미, 문자 등 형태와 형식이 정해져 있지 않음

정량적 데이터 (Quantitative data): 수치, 길이, 도형으로 표시

출제 유형

  • 정성적 및 정량적 데이터 구분 문제

03 암묵지 vs 형식지

데이터는 지식경영의 핵심 이슈인 암묵지와 형식지의 상호 작용에 있어 중요한 역할을 함.

출제 유형

  • 형식지와 암묵지 개념과 사례 구분

04 암묵지와 형식지의 상호 작용

  • 공통화 (Socialization)
  • 표출화 (Externalization)
  • 연결화 (Combination)
  • 내면화 (Internalization)

출제 유형

  • 상호작용의 4가지 개념

05 데이터와 정보의 관계

데이터의 담위적 특성이 근거의 기능에 주목하여 데이터와 정보의 관계를 살펴봄.

DIKW 피라미드 (Data → Information → Knowledge → Wisdom)

출제 유형

  • DIKW 개념 구분 사례

06 데이터베이스 정의

동시에 복수의 적용 업무를 지원할 수 있도록 설계된 데이터 저장소.

출제 유형

  • 데이터베이스와 데이터베이스 관리시스템 개념 구분

07 DBMS의 발전 과정

1세대: 네트워크 DBMS, 계층 DBMS

2세대: 관계형 DBMS

3세대: 객체 지향형 DBMS

08 관계형 데이터베이스 vs 객체 지향형 데이터베이스

관계형 데이터베이스 (RDBMS)와 객체 지향형 데이터베이스 (ODBMS)의 차이점

출제 유형

  • RDBMS와 ODBMS 차이점

09 데이터 유형 분류

정형 데이터, 반정형 데이터, 비정형 데이터의 특징과 종류

출제 유형

  • 데이터의 유형 특성 구분, 종류

10 스키마와 인스턴스

스키마 (Schema): 구조를 만드는 것

인스턴스 (Instance): 특정 시점의 데이터베이스 내용

11 데이터베이스 설계 순서

  1. 요구조건 분석
  2. 개념적 설계
  3. 논리적 설계
  4. 물리적 설계

12 데이터베이스의 특징

통합된 데이터, 저장된 데이터, 공유된 데이터, 변경되는 데이터

13 데이터베이스 특성

  • 정보의 축적 및 전달 측면
  • 정보의 보관 측면
  • 정보 관리 측면
  • 정보 가공 측면
  • 정보 활용 측면

06 데이터베이스 정의

동시에 복수의 적용 업무를 지원할 수 있도록 복수의 사용자 요구에 대응하여 저장, 공급하기 위해 일정한 구조에 따라 생성된 데이터 집합.

데이터베이스 관리 시스템(DBMS): 데이터베이스를 관리하고 공유할 수 있도록 제공하는 소프트웨어.

출제 유형

  • 데이터베이스와 DBMS의 개념 구분

07 DBMS의 발전 과정

  • 1세대: 네트워크 DBMS, 계층 DBMS
  • 2세대: 관계형 DBMS (예: Oracle, MySQL)
  • 3세대: 객체 지향형 DBMS (Objected DBMS)

08 관계형 데이터베이스 (RDBMS) vs 객체 지향형 데이터베이스 (ODBMS)

구분 관계형 데이터베이스 (RDBMS) 객체 지향형 데이터베이스 (ODBMS)
데이터 타입 문자, 숫자, 날짜와 같은 단순한 데이터 타입 지원 사용자 정의 타입 및 복잡한 데이터 타입 지원
주된 장점 오랜 기간에 걸쳐 검증된 안정성과 대규모 데이터 처리 성능 복잡한 데이터 구조의 모델링 가능
주된 단점 복잡한 데이터 구조를 표현하기 어려움 기존의 관계형 데이터베이스에 비해 성능이 낮을 수 있음

출제 유형

  • RDBMS와 ODBMS의 차이점
